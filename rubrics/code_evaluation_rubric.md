# Code Evaluation Rubric

Score each model answer from 1 to 5 in each category.

## Correctness

- 5: Fully satisfies every stated requirement.
- 3: Works for common cases but misses edge cases.
- 1: Fails core requirements or crashes.

## Reliability

- 5: Handles errors, invalid inputs, and lifecycle concerns.
- 3: Handles some errors but leaves important gaps.
- 1: Panics, crashes, leaks state, or ignores failure modes.

## Efficiency

- 5: Uses appropriate time/space complexity.
- 3: Acceptable for small inputs but poor at scale.
- 1: Inefficient or unbounded in obvious ways.

## Maintainability

- 5: Simple, readable, testable, and idiomatic.
- 3: Understandable but coupled or awkward.
- 1: Hard to reason about or modify safely.

## Testing

- 5: Covers success, failure, and edge cases.
- 3: Covers basic success cases only.
- 1: No meaningful tests.

## Written Rationale

A strong evaluation explains what is wrong, why it matters, and how the fix addresses it. Prefer concrete examples over vague criticism.
