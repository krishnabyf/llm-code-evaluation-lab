# Evaluation Summary

This lab covers six common LLM code evaluation patterns:

- Python: state management, time-window correctness, memory cleanup.
- JavaScript/React: stale async state and abortable requests.
- Java: data-structure design and O(1) cache behavior.
- C++: algorithm correctness, duplicate handling, sentinel output.
- Go: error handling and retry semantics.
- Rust: Result-based parsing instead of panic-driven code.

Each task includes an intentionally flawed answer, a corrected implementation, tests, and a written evaluation. This mirrors the day-to-day work of building coding datasets, reviewing model output, and designing automatic verification.
